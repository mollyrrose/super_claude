# 标题生成

为文章生成多个候选标题。

## 执行流程

### 1. 理解文章

读取用户提供的文章内容（或核心观点摘要），提取：
- 核心观点（一句话）
- 情绪基调（理性/感性/犀利/温和）
- 目标受众（技术人/普通读者/创业者等）
- 关键冲突或反差（如果有）

### 2. 加载标题偏好

尝试读取 `./writing-workspace/styles/my_style.json`。

- 如果存在：参考其中的 `structure.title_pattern` 偏好，将该类型标题排在候选列表前面
- 如果不存在：不影响生成，跳过此步

### 3. 生成候选标题

生成 5-8 个候选标题，覆盖以下策略：

| 策略 | 特点 | 示例 |
|------|------|------|
| 数字型 | 用数字制造具体感 | "被忽略的3个AI真相" |
| 反常识型 | 挑战普遍认知 | "效率最高的人从不列清单" |
| 痛点型 | 戳中读者困扰 | "为什么你学了那么多，还是不会写" |
| 悬念型 | 制造好奇心 | "看完这个实验，我重新理解了焦虑" |
| 直给型 | 简洁有力地陈述 | "程序员不会被AI取代" |
| 提问型 | 用问题引发思考 | "你上一次改变想法是什么时候？" |
| 对比型 | 用反差制造张力 | "AI写的文章越来越好，但你不该高兴" |

不需要每种策略都覆盖，选择最适合文章内容的 5-8 种。

### 4. 标注信息

为每个标题标注：
- **策略类型**
- **适用平台**：公众号（可以长一些）、小红书（要口语化吸睛）、Twitter（要简短）、博客（可以严肃正式）
- **语气**：如果和文章基调特别匹配，标记"推荐"

### 5. 展示结果

```
## 候选标题

文章核心：[一句话概括]

| # | 标题 | 策略 | 适合平台 | 备注 |
|---|------|------|---------|------|
| 1 | 为什么大多数人学不会"独立思考" | 痛点型 | 公众号 | ⭐ 推荐 |
| 2 | 你上一次改变想法是什么时候？ | 提问型 | 公众号/Twitter | |
| 3 | 独立思考的3个反直觉真相 | 数字+反常识 | 公众号/小红书 | |
| 4 | 独立思考不会让你更快乐 | 反常识型 | 公众号 | |
| 5 | 所有人都在"思考"，但没人在思考 | 对比型 | Twitter/公众号 | |
| 6 | 别急着转发——你可能只是在找认同感 | 悬念型 | 小红书 | |

选一个你喜欢的，我可以在这个基础上微调变体。
```

## 注意事项

- **避免标题党**：标题要有吸引力但不能夸大或误导。文章里没有的内容不要写进标题。
- **长度差异**：公众号标题可以 15-25 字，Twitter 尽量 15 字以内，小红书 10-20 字。
- **不要太安全**：太平淡的标题（如"关于独立思考的一些想法"）不值得列出来。每个标题都应该让人想点进去。
- **微调服务**：如果用户选了某个标题但想调整，基于该标题生成 3 个变体。
